Python Virtual Environment

What is a Virtual Environment?

A virtual environment is an isolated Python environment that allows you to:

  • Install packages without affecting system Python
  • Manage different project dependencies separately
  • Avoid version conflicts between projects
  • Create reproducible development environments

Important

Best Practice: Always use a virtual environment for your projects!

Creating Virtual Environments

Using venv (Built-in)

2. Control Flow & Logic

If / Elif / Else

temperature = 25

if temperature > 30:
    print("It's hot!")
elif temperature > 20:
    print("It's warm")
elif temperature > 10:
    print("It's cool")
else:
    print("It's cold!")

For Loops

# Loop through a list
fruits = ["apple", "banana", "orange"]
for fruit in fruits:
    print(fruit)

# Loop through a range
for i in range(5):
    print(i)  # 0, 1, 2, 3, 4

# Loop with index
for i, fruit in enumerate(fruits):
    print(f"{i}: {fruit}")

While Loops

# Basic while loop
count = 0
while count < 5:
    print(count)
    count += 1

# While with condition
password = ""
while password != "python":
    password = input("Enter password: ")
    
print("Access granted!")

Range Function

# range(stop)
range(5)           # 0, 1, 2, 3, 4

# range(start, stop)
range(2, 7)        # 2, 3, 4, 5, 6

# range(start, stop, step)
range(0, 10, 2)    # 0, 2, 4, 6, 8

# Reverse range
range(10, 0, -1)   # 10, 9, 8, ..., 1

Break and Continue

# Break - exit loop
for i in range(10):
    if i == 5:
        break
    print(i)  # 0, 1, 2, 3, 4

# Continue - skip iteration
for i in range(5):
    if i == 2:
        continue
    print(i)  # 0, 1, 3, 4

4. Functions & Modularity

Defining Functions

# Basic function
def greet():
    print("Hello!")

def greet_person(name):
    print(f"Hello, {name}!")

Parameters & Return Values

# Function with return
def add(a, b):
    return a + b

result = add(5, 3)  # result = 8

# Multiple returns
def get_min_max(numbers):
    return min(numbers), max(numbers)

minimum, maximum = get_min_max([1, 5, 3, 9, 2])

Default Parameters

def greet(name, greeting="Hello"):
    return f"{greeting}, {name}!"

# Using default
greet("Alice")              # "Hello, Alice!"

# Overriding default
greet("Bob", "Hi")          # "Hi, Bob!"
greet("Charlie", greeting="Hey")  # "Hey, Charlie!"

Scope - Local vs Global

# Global variable
x = 10

def example():
    # Local variable
    y = 5
    print(x)  # Can access global
    print(y)  # Can access local

example()
print(y)  # ERROR: y is not defined

# Modifying global
def modify_global():
    global x
    x = 20

modify_global()
print(x)  # 20

Lambda Functions

# Regular function
def square(x):
    return x ** 2

# Lambda (anonymous function)
square = lambda x: x ** 2

# Common use with map/filter
numbers = [1, 2, 3, 4, 5]
squared = list(map(lambda x: x ** 2, numbers))
# [1, 4, 9, 16, 25]

evens = list(filter(lambda x: x % 2 == 0, numbers))
# [2, 4]

7. Error Handling & Debugging

Common Python Errors

# SyntaxError
print("Hello"  # Missing closing parenthesis

# NameError
print(x)  # x is not defined

# TypeError
"5" + 5  # Can't add string and int

# IndexError
my_list = [1, 2, 3]
my_list[10]  # Index out of range

# KeyError
my_dict = {"name": "Alice"}
my_dict["age"]  # Key doesn't exist

Try / Except

# Basic exception handling
try:
    number = int(input("Enter a number: "))
    result = 10 / number
    print(f"Result: {result}")
except ValueError:
    print("Invalid input! Please enter a number.")
except ZeroDivisionError:
    print("Cannot divide by zero!")

# Catch any exception
try:
    # risky code
    pass
except Exception as e:
    print(f"An error occurred: {e}")

Try / Except / Else / Finally

try:
    file = open('data.txt', 'r')
    content = file.read()
except FileNotFoundError:
    print("File not found!")
else:
    # Runs if no exception occurred
    print("File read successfully!")
finally:
    # Always runs, even if error occurs
    try:
        file.close()
    except:
        pass
    print("Cleanup complete")

Writing Safe Code

# Bad: Assumes file exists
file = open('data.txt')
content = file.read()
file.close()

# Good: Handles potential errors
try:
    with open('data.txt', 'r') as file:
        content = file.read()
except FileNotFoundError:
    print("File doesn't exist")
    content = None

# Bad: Assumes valid input
age = int(input("Age: "))

# Good: Validates input
while True:
    try:
        age = int(input("Age: "))
        break
    except ValueError:
        print("Please enter a valid number")

Debugging Mindset

  1. Read the error message carefully
    • What type of error?
    • Which line number?
  2. Check your assumptions
    • Print variable values
    • Verify data types
  3. Isolate the problem
    • Comment out code sections
    • Test small pieces
  4. Use print() debugging
    • Add print statements
    • Track variable values
  5. Google the error
    • Copy exact error message
    • Check Stack Overflow

Debugging Example

# Bug: IndexError

    for i in range(len(numbers) + 1):  # BUG!

def get_average(numbers):

9. Code Quality & Best Practices

Naming Conventions

# Variables and functions: snake_case
user_name = "Alice"
def calculate_average(numbers):
    pass

# Constants: UPPER_CASE
MAX_SIZE = 100
PI = 3.14159

# Classes: PascalCase
class UserAccount:
    pass

# Private (by convention): _leading_underscore
_internal_variable = 42

# Avoid
x = 5  # Too short
thisIsJavaScript = True  # Wrong style

Indentation

# Good: 4 spaces (PEP 8 standard)
def calculate_sum(numbers):
    total = 0
    for num in numbers:
        total += num
    return total

# Bad: Inconsistent indentation
def calculate_sum(numbers):
   total = 0
      for num in numbers:  # Wrong indentation
        total += num
    return total

# Bad: Mixing tabs and spaces
def calculate_sum(numbers):
    total = 0        # Tab
    for num in numbers:  # Spaces

Comments & Docstrings

# Single-line comments
x = 5  # Initialize counter

# Multi-line comments
"""
This is a multi-line comment.
It can span several lines.
"""

# Function docstrings
def calculate_average(numbers):
    """
    Calculate the average of a list of numbers.
    
    Args:
        numbers (list): List of numbers
        
    Returns:
        float: The average value
    """
    return sum(numbers) / len(numbers)

Writing Readable Code

# Bad: Hard to read
def f(l):
    t=0
    for i in l:
        t+=i
    return t/len(l)

# Good: Clear and readable
def calculate_average(numbers):
    """Calculate the average of a list of numbers."""
    total = sum(numbers)
    count = len(numbers)
    return total / count

# Use descriptive names
# Break complex logic into steps
# Add comments for complex operations

Code Organization

# Good structure
# 1. Imports
import math
from datetime import datetime

# 2. Constants
MAX_USERS = 100
PI = 3.14159

# 3. Functions
def helper_function():
    pass

def main_function():
    pass

# 4. Main execution
if __name__ == "__main__":
    main_function()

PEP 8 Style Guide

  • Line length: Maximum 79 characters
  • Blank lines: 2 before functions, 1 inside
  • Spaces: Around operators (x = 5, not x=5)
  • Imports: One per line, grouped
  • Strings: Use double quotes for docstrings
  • Avoid: Multiple statements on one line
# Good
x = 5
y = 10

# Bad
x=5;y=10

Common Best Practices

  1. DRY: Don’t Repeat Yourself
  2. KISS: Keep It Simple, Stupid
  3. Use meaningful variable names
  4. Write functions for reusable code
  5. Comment why, not what
  6. Handle errors gracefully
  7. Test your code

Exception Handling

Try-Except Blocks

Try-Except-Else-Finally

try:
    file = open('data.txt', 'r')
    content = file.read()
except FileNotFoundError:
    print("File not found!")
else:
    # Runs if no exception occurs
    print("File read successfully")
    print(content)
finally:
    # Always runs, even if exception occurs
    try:
        file.close()
    except:
        pass
    print("Cleanup complete")

Common Pitfalls

Mutable Default Arguments (Avoid):

# Wrong
def append_to(item, lst=[]):
    lst.append(item)
    return lst

# Right
def append_to(item, lst=None):
    if lst is None:
        lst = []
    lst.append(item)
    return lst

Modifying List While Iterating (Avoid):

# Wrong
for item in my_list:
    if condition:
        my_list.remove(item)

# Right
my_list = [x for x in my_list 
           if not condition]
